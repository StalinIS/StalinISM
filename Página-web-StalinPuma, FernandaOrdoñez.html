Problema

Monitorear el estado de funcionamiento de 20 máquinas en el colegio Unidad Educativa Remigio Romero y Cordero.

Preguntas de investigación

1. Metas y Objetivos

Implementar un sistema de monitoreo para garantizar la disponibilidad y eficiencia de las máquinas, reducir el tiempo de inactividad y mejorar la productividad.

2. Beneficiarios

  • Administración del colegio:

    Facilita la gestión de los recursos técnicos, permitiendo una planificación efectiva de reparaciones y adquisiciones futuras. También ayuda a identificar patrones de uso, lo que permite asignar presupuestos de manera más eficiente.

  • Profesores:

    Permite a los docentes programar sus clases y actividades en base a la disponibilidad de máquinas, evitando interrupciones inesperadas. Además, se les proporciona acceso a informes que muestran el estado de los equipos, permitiendo planificar mejor las lecciones prácticas.

  • Estudiantes:

    Mejora la experiencia educativa al garantizar la disponibilidad de equipos funcionales para prácticas y proyectos. El sistema también puede integrarse con herramientas de reserva para que los estudiantes gestionen el uso de las máquinas de manera eficiente. el historial de mantenimiento ayuda a realizar diagnósticos más rápidos y precisos.

  • Personal de mantenimiento:

    Proporciona alertas automáticas sobre problemas de las máquinas, lo que permite realizar mantenimientos preventivos antes de que ocurra una falla mayor.

3. Objetivo Principal

Desarrollar un sistema web avanzado que permita monitorear en tiempo real el es tado de funcionamiento de las máquinas en diversas áreas del colegio, proporcion ando notificaciones inmediatas en caso de fallas o averías. Este sistema también re gistrará el historial de uso de cada máquina, permitiendo realizar análisis predicti vos sobre su vida útil y la necesidad de mantenimientos futuros. A través de este sistema, se podrán generar informes detallados que incluyan métricas clave como el tiempo de actividad, el uso promedio y los costos asociados a reparaciones.

4. Métodos

  • Investigación documental:

    Recopilación de información sobre mejores prácticas en la gestión de mantenimiento de equipos, incluyendo estudios sobre monitoreo remoto de máquinas y su impacto en la eficiencia operativa. También se analizarán casos de éxito en la implementación de sistemas similares.

  • Análisis de requerimientos:

    Reuniones con los diferentes grupos de usuarios (administración, profesores, estudiantes y personal de mantenimiento) para entender sus necesidades específicas. Se priorizarán las funcionalidades que faciliten la operación diaria y la reducción de costos por fallos inesperados.

  • Diseño de interfaz de usuario:

    Proceso iterativo de diseño en el que se crearán múltiples prototipos, seguidos de pruebas de usabilidad con usuarios clave. La interfaz estará optimizada para dispositivos móviles y se integrará con herramientas de visualización de datos para mostrar gráficas de rendimiento y alertas en tiempo real.

  • Desarrollo web:

    Utilización de tecnologías como Node.js, Express para el backend, y React para el frontend, garantizando una arquitectura escalable. Se integrará con sensores IoT conectados a las máquinas para enviar datos en tiempo real al servidor, donde se almacenarán en una base de datos NoSQL como MongoDB. Además, se desarrollarán APIs RESTful para interactuar con las interfaces de usuario y otros sistemas externos.

  • Pruebas y depuración:

    Además de las pruebas unitarias y de integración, se realizarán pruebas de rendimiento bajo distintas cargas de trabajo para asegurar que el sistema pueda manejar el monitoreo de múltiples máquinas simultáneamente. También se realizarán pruebas de seguridad para proteger la integridad de los datos y evitar accesos no autorizados.

Solución Propuesta

Desarrollar un sistema web de monitoreo utilizando tecnologías como React, Node.js y MongoDB. Crear una interfaz de usuario intuitiva para registrar y visual izar el estado de las máquinas. Implementar notificaciones automáticas para informar sobre problemas de funcionamiento.
